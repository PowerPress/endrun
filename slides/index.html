Endrun

Secure Digital Communications For Our Modern Dystopia

By Grant Dobbe and Brendan O'Connor

The Takeaways

No Internet Use

We're making an end run around the Internet. Because someone has to.

Endrun is designed to make an ``end run'' around the Internet, for the purpose of giving a communications option to people who cannot use the Internet. This could happen for any of several reasons: the Internet could have suffered a major regional or global failure, due to natural disaster, accident, or hostility; the Internet might be considered untrustworthy, due to adversary action; or the areas being communicated across might be so remote that Internet connectivity is simply not a useful option.

By Any Means Necessary

  • Sneakernet
  • XBee
  • Sneakernet+
  • Carrier Pigeon
  • Amateur Radio*
Endrun is "by any means necessary." Endrun is designed to work across any form of communication that is available, whether that's physical data movement and low-power transient unlicensed radio links, as we would recommend, or by Earth-Moon-Earth (EME) high-powered signals from fix points, or even amateur (ham) radio. The only firm requirement for Endrun is a USB port; as we'll see later, that's more powerful than you might think.

Old-School Cellular Communications

Cell as in "group", not cell as in "for the enjoyment of all patrons, please turn off your"

Endrun is designed for communications between small groups of people---``cells'' or ``pods.'' It is not designed for open-ended communication between people that have no prior relationship.

Disruption/Delay-Tolerant

  • Conflict zones
  • Disaster areas
  • Hostile terrain (inc. N. Scotland, Scandinavia, the southern U.S.)
  • SPAAAAAAAAAACE!
Endrun is disruption- and delay-tolerant. It is designed to work where nodes have no real- or bounded-time connection to each other, for instance, when couriers are moving data stored on physical media either around town or across a continent.

Modifiable On the Fly

This isn't an Exchange Server. You know your needs better than we do.

Endrun is built using standard, well-tested, open source components and combined using Python, rather than using all custom code. This lets Endrun benefit from decades of work from thousands of programmers (with all the attendant work on security that implies). It also means that Endrun is customizable by programmers in the field; in an emergency situation, this can give flexibility that fully-custom solutions cannot.

Previous Work

Tor

The choice of dissidents and activists everywhere.
Add notes here. Basic idea: Tor provides privacy by encrypting communication at every relay using different keys. However, exit nodes are the weakpoint, and an adversary with sufficient resources could control the majority of end nodes.

AirChat, by Anonymous

Internet-free, end-to-end encrypted text and low-bandwidth voice communication
Airchat provides secure encrypted messaging, but uses amateur radio and is illegal in the US under part 97 of the FCC rules. Other countries have similar restrictions, so it's illegal there too. It also relies on semi-permanent high-power radio links, which means anyone with a couple of directional antennas can ruin your day by calling in air support, or as we call it, "connection reset by missile".

Commotion Wireless / Project Byzantium

Neighborhood mesh networks for the Zombie apocalypse
Commotion Wireless creates ad-hoc mesh networks using regular 802.11a/b/g/n hardware. Pretty cool. In a "zombie apocalypse", this would work pretty well. However, wifi is semi-permanent wireless radio. Wifi has the same problems we described above.

NASA/ESA Delay-Tolerant Networking

Close, but no appropriations
DTN has been around since the early 90s when Vint Cerf stated working on the InterPlaNet. It solves most of the problems we described above: it'll handle long delays, transient links, multiple transport paths, etc. However, security is a second-tier concern if it's considered at all. Also, for the most part, DTN has been an academic exercise, limited to university research teams with grant funding from NASA or ESA. It's never been implemented the same way twice, and the closest we've come to having a standard implementation of it -- in the form of NASA's proposed Mars Telecommunications Orbiter -- fell victim to the Appropriations Committee of the United States Congress.

Endrun 101

Ok, now we get into the particulars. Like we said, DTN is the closest existing concept to what we want to do. However, it's never been implemented the same way twice, and it has security holes you could drive a boat through. So, using existing software, we put together our own secure implementation of the DTN protocol using:

  • Git
  • LibSodium
  • Some Python to glue everything together
And called it Endrun.

Git

Git solves many of the problems that a DTN faces.

  • Information order
  • Information integrity
  • Information divergence

There are a lot of problems with putting together a networking protocol; stuff shows up in the wrong order, stuff shows up mangled, stuff... doesn't show up, stuff shows up twice. Same goes for DTN. However, because of the latency concerns, we can't just throw TCP at it and call it good enough.

It turns out this problem already has a solution. Git resolves:

  • Information showing up in the wrong order

Git Bundles

Bundles let us "fake" remote repos.

From the man page:

	    This command provides support for git fetch and git pull 
	    to operate by packaging objects and references in an 
	    archive at the originating machine, then importing those 
	    into another repository using git fetch and git pull after 
	    moving the archive by some means.

Look, ma! No network connection!

Git bundles are a way of using a file to connect to a remote repo instead of a network connection. They were originally intended for air-gap situations and contexts in which people have either high latency or low bandwidth. We have all three, so naturally, it's a perfect fit.

NaCl

NaCl is a cryptography suite that was developed by Daniel J. Bernstein of the University of Chicago.

  • Bleeding-edge elliptical-curve cryptography

  • Fast and efficient
  • Unfortunately, not guaranteed to run anywhere but DJB's office

LibSodium

LibSodium is a wrapper around NaCl developed by the friendly folks at OpenDNS.

  • Has bindings in a number of different languages (Python, Ruby, PHP, etc.)
  • Much easier to work with

Endrun uses LibSodium to sign, encrypt, and sign bundles before they are added to Endrun payloads.

Payload Format

Payloads are how Endrun moves data around. Each payload contains:

  • Origin node identifier
  • Destination node identifier
  • Timestamp (TTL)
  • Contents
  • Chain of Custody data

Routing

Pairwise Communication

DTN routing occurs between pairs of nodes.

Imagine a bucket brigade.

Spray and Pray Wait

  • Epidemic routing uses too much bandwidth.
  • Other routing methods can't handle transient links.
  • Spray and Wait combines store-and-forward with a limit on message copies.
We use a routing algorithm referred to by its designers as "spray-and-wait" as a way to get the best of store-and-forward routing (hold it until you can move it) without flooding a network with tons of packets (which, if you're using radio, means almost constant radio traffic).
